# Project Baseline Review

- Inspect the full repository from root, not just the open solution.
- Identify current-state and target-state stacks.
- Produce durable project overview, solution map, dependency map, and skill recommendations.

## Caveman Mode

Use **caveman-lite** for all baseline outputs. Drop filler; keep sentence structure and full technical scope.

- solution-map: terse bullet list per service/layer — no prose introduction
- dependency-map: direct table format — no explanatory preamble
- skill-recommendations: `<skill>: <reason one sentence>` per entry
