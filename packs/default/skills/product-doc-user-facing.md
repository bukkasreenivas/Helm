# Product Doc User Facing

- Write from the user perspective.
- Focus on value, workflow, permissions, limitations, and notable behavior changes.
- Avoid internal implementation detail unless it affects usage.

## Caveman Mode

Use **caveman-lite** for product doc drafts. Drop hedging and filler; keep sentence structure and user clarity.

- Lead with the value, not the background: "Users can now X" not "As part of this release, we have added the ability to..."
- Limitations: short bullet list, direct — no softening language
